# 제품 요구사항 정의서 (PRD)

## 1. 제품 개요
- **제품명**: Structure & Protection Topology Lab
- **부제**: Cantilever Beam 위상최적화 및 AI 코딩 실습
- **분류**: 교육용 엔지니어링 시뮬레이션 웹 애플리케이션
- **플랫폼**: 웹 브라우저 (Google AI Studio Build mode 기반)

## 2. 문제 정의
방산업체 구조·방호 직무 신입사원들은 방대한 상용 CAE 프로그램의 조작법을 배우기 전에, 실제 유한요소해석(FEA)과 위상최적화의 수학적 배경 및 알고리즘의 동작 원리를 이해할 필요가 있습니다. 또한, AI 코딩 도구를 활용하여 반복적인 해석 업무를 자동화하는 스크립트를 작성하는 능력을 배양해야 하지만, 실제 업무 데이터를 사용할 수 없으므로 안전하고 직관적인 교육용 샌드박스 환경이 필요합니다.

## 3. 개발 목적
- 상용 CAE를 대체하는 것이 아니라, FEA와 최적화의 원리를 코드 레벨에서 이해하기 위함
- 설계영역, 하중, 경계조건이 수치 모델로 변환되는 과정을 시각화
- 반복 계산을 자동화하는 논리를 체득하고, AI 코딩 도구에 기술 요구사항을 정확하게 전달하는 방법 학습
- 검증된 수치해석 결과를 통해 오류를 식별하고, 이론적 결과와 비교하는 능력 배양

## 4. 핵심 사용자
- 구조·방호 직무에 배치된 신입 기계 엔지니어
- 위상최적화 및 수치해석 기초를 학습하려는 학생 및 연구원

## 5. 사용자 시나리오
1. 신입사원이 앱에 접속하여 기본 제공되는 Cantilever Beam 예제를 확인합니다.
2. 재료 물성(Young's modulus, Poisson's ratio)과 형상 정보, 체적분율 제약을 설정합니다.
3. '최적화 실행' 버튼을 눌러 브라우저에서 실시간으로 계산되는 위상최적화 과정을 시각적으로 확인합니다.
4. 계산 중 일시정지하여 현재 반복(Iteration)의 변위와 응력 분포를 분석합니다.
5. 최적화가 완료된 후, 결과 요약을 확인하고 변위-컴플라이언스 그래프를 통해 수렴성을 검토합니다.
6. 결과를 JSON, PNG 형태 및 CSV로 내보내고, Python 기준 코드와 결과를 교차 검증합니다.
7. '코드 학습' 탭을 통해 입력값이 어떻게 FEA 행렬로 변환되고 풀리는지 단계별로 매핑된 파일 구조를 파악합니다.

## 6. 기능 요구사항
- **시뮬레이션 제어**: 실행, 일시정지, 1 Iteration 단위 실행, 중지, 초기화, 기본값 복원 기능
- **결과 시각화**: 
  - 밀도(Density) 분포 (Grayscale/Colormap, 격자, Threshold 적용)
  - 변위(Displacement) (변형 전/후 중첩, Colormap, 배율 조절)
  - 폰 미세스 응력(Von Mises Stress) 분포 (후처리 결과임에 대한 명시적 경고 포함)
  - 요소망(Mesh) 상태, 하중 및 구속조건 가시화
- **그래프**: Iteration 대비 Compliance, Volume fraction, Density change, Residual, Time
- **비교 분석**: 최적화 전(Full-density)과 후의 체적, 질량, 변위, 응력, 강성 대비 질량 지표 비교
- **내보내기**: PNG, JSON(결과 및 설정), CSV(이력), Markdown(요약) 내보내기/가져오기 기능
- **교육 기능**: 각 해석 단계별 코드 매핑(예: mesh 생성 -> `mesh.ts`) 및 이론 설명 제공

## 7. 비기능 요구사항
- **성능**: 최대 5,000 요소 이하에서 UI 프리징 없이 실시간 계산(Web Worker 사용)
- **접근성**: 키보드 네비게이션 및 명확한 오류 피드백, 한국어 인터페이스 적용
- **안정성**: TypeScript Strict mode 적용, 입력값 유효성 검증, 수치적 불안정성 대응(예: 특이 행렬 경고)
- **보안**: 모든 계산은 로컬(브라우저)에서 수행, 외부 서버 통신 없음, 방산 데이터 미사용

## 8. 수치해석 모델
- 2차원 평면응력(Plane Stress), 선형탄성, 등방성 재료 가정
- 4절점 사각형 요소 (Q4 Element), 절점당 2자유도
- SIMP (Solid Isotropic Material with Penalization) 모델 적용 ($E = E_{min} + \rho^p(E_0 - E_{min})$)

## 9. 위상최적화 알고리즘
- 최소 컴플라이언스(Minimum Compliance) 목적함수
- 체적분율 제약조건 (Volume Fraction Constraint)
- 밀도 기반 최적화(SIMP density method) 및 Optimality Criteria (OC) 업데이트
- 밀도 필터 적용(Density Filter) 및 이동 한계(Move limit) 제어

## 10. UI/UX 요구사항
- 상단: 제목 및 전역 경고 배너
- 좌측: 아코디언 형태의 설정 패널 (모델, 재료, 하중, 최적화 변수)
- 중앙: 실시간 시각화 Canvas (밀도, 변위, 응력, 요소망 탭 분리)
- 우측/하단: 실시간 수렴 지표, 차트, 결과 요약 및 교육용 탭 배치
- 명확한 공학 단위(mm, N, MPa) 명시 및 엔지니어링 대시보드 테마 적용

## 11. 시스템 아키텍처
- 프론트엔드: React 19, Vite, Tailwind CSS
- 상태 관리: React Hooks 기반 커스텀 상태 머신
- 백그라운드 연산: Web Worker (`topologyWorker.ts`)로 UI 스레드 분리
- 선형 방정식 풀이: 직접 구현한 PCG (Preconditioned Conjugate Gradient) 희소 행렬 솔버

## 12. 데이터 구조
- `InputState`: 형상, 재료, 하중, 최적화 파라미터 (JSON 직렬화 가능)
- `MeshState`: 절점 좌표, 요소 연결, 자유도 매핑
- `FEAState`: 전역 강성 행렬, 변위 벡터, 응력 배열
- `OptimizationState`: 요소별 밀도(설계변수, 물리밀도), Iteration 기록(Compliance 등)

## 13. 검증 및 시험계획
- **단위 테스트**: 요소 강성행렬 대칭성, PCG 솔버 정확성, 밀도 필터 적용 등 단위 모듈 테스트
- **통합 검증**: 최종 체적분율 오차 확인($\pm 0.005$), 초기 Compliance 일치 여부 확인
- **이론 비교**: 슬렌더 빔(Slender beam) 조건 시 빔 이론과 FEA 변위 결과 비교(참고용)

## 14. 오류 처리
- 해석 발산 시(예: PCG Residual 발산) 계산 중지 및 경고 알림
- 요소 수 5,000개 초과 설정 시 성능 경고 및 확인 요구
- 잘못된 단위나 음수 입력 시 실시간 입력 폼 검증

## 15. 보안 및 데이터 취급 원칙
- 본 앱은 훈련 및 교육용 예제 환경이므로, 실제 방산 프로젝트 데이터를 절대 입력하지 않도록 경고문 지속 노출
- 계산 결과는 사용자의 로컬 환경에만 임시 저장되며, 앱 종료 시 소멸됨

## 16. MVP 범위
- Cantilever Beam 기반 선형 정적 2D 최소 컴플라이언스 최적화
- 브라우저 내 시각화 및 Web Worker 연산
- 로컬 파일 내보내기/가져오기
- Python 기준 코드 제공

## 17. 비범위 사항
- 3D 해석, 소성, 대변형, 동적 하중, 열하중, 충격/폭발 해석
- 외부 데이터베이스 연동, 사용자 로그인, 서버 사이드 연산
- 응력 제약 기반 최적화 알고리즘

## 18. 향후 확장 로드맵
- 다중 하중 조건(Multiple Load Cases) 지원
- 다양한 경계조건 시나리오(예: 브래킷, MBB Beam) 템플릿 추가
- 제조 제약조건(압출 방향, 대칭 등) 적용 옵션

## 19. 완료 조건과 Acceptance Criteria
- 입력된 기본값으로 앱을 실행 시 최적화 과정이 정상적으로 시각화되고 종료되어야 함
- Web Worker를 통해 시각화 중에도 UI가 멈추지 않아야 함
- 결과 내보내기(JSON, CSV)가 정상 동작해야 함
- 앱 내에 실제 방산 데이터 사용 금지 경고문이 상시 노출되어야 함
- Python 기준 구현 코드와 주요 결과가 상호 검증 가능한 수준이어야 함
